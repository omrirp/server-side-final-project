מחלקות:
  • המחלקות הן: Apartment, Host, Reservation, Review ו User
  • מחלקת Reservation מחזיקה מופע של אובייקט מסוג Apartment שמחזיקה מופע של אובייקט שלם מסוג Host
    יתרונות: כיוון שכאשר משתמש רוצה לצפות בפרטי הדירה המלאים גם בחיפוש דירות וגם בהזמנות שבוצעו מפני שכל המידע הרלוונטי כבר נמצא, רנדור הנתונים המלאים מהיר יותר. כמו כן כאשר מחפשים דירה לחופשה רוב המשתמשים משתמשים בחיפוש מתקדם לפי פרמטרים מה שגורם לצד הלקוח לא לשלוף יותר מדי נתונים מהשרת.
    חסרונות: בחיפוש רגיל לפי תאריכים בלבד צד הלקוח באמת יכול להביא המון נתונים מה שגורם לחיפוש להוית איטי יותר ועמוס במידע. בנוסף לממשק המנהל כאשר רוצה להציג את הפרטים שנתבקשנו הוא שולף גם נתונים מיותרים אבל עם זאת אפשר להשתמש בהם בעתיד אם נחשוב שהאתר הזה באמת משומש.
